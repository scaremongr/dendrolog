# DendroLog — Краткая справка

Быстрый просмотрщик больших лог-файлов: вкладки с несколькими файлами,
извлечение полей, фильтрация, подсветка и дочитывание «на лету».

> Подсказка: любая горячая клавиша ниже настраивается в **Tools → Settings → Shortcuts**.

---

## Открытие логов

- **Открыть файл(ы)** — `File → Open` (`Ctrl+O`). Несколько файлов открываются в одной вкладке и сливаются по времени.
- Панель **Directory Scanner** — поиск лог-файлов в папке по расширениям и открытие одного или нескольких сразу.
- **Recent Files** — `File → Recent Files`.
- **Save View As** — `File → Save View As…` (`Ctrl+Shift+S`). Сохраняет именно то, что сейчас видно во view (активные фильтры **и** выбор Log Fields) в **новый** файл. Открытые файлы не перезаписываются.
- Список расширений (например `log, txt`) общий для открытия, сохранения и сканера — задаётся в **Settings → General**.

## Чтение представления

- **Вкладки** — каждая вкладка независима; фильтры/маркеры свои у каждой.
- **Маркер `›` в желобке** — начало логической записи; зелёный — строки из последнего авто-дочитывания.
- **Плашки справа:**
  - Плашка файла — цвет источника (когда во вкладке несколько файлов).
  - `+N` / `−` — строка длиннее видимой области; клик разворачивает/сворачивает её.
- **Подсветка синтаксиса** — строки, числа, hex, URL, пути, GUID, таймстампы, парные скобки.
- **Перенос по словам (Word Wrap)** — кнопка на панели, `View → Word Wrap` или `Alt+Z` (значение по умолчанию — в **Settings → View**).

## Выделение, копирование и действия по контексту

- **Протяжка** мышью выделяет текст; **двойной клик** — умный токен.
- **Двойной клик распознаёт:** строки в кавычках, таймстампы, URL, пути к файлам, hex-литералы, IP-адреса, имена файлов, числа, слова. Подряд идущие разделители выделяются группой; клик по пробелам выделяет только пробелы (без перескока на следующее слово).
- **Расширение выделения с клавиатуры:** `Shift+←/→` — на символ, `Ctrl+Shift+←/→` — на токен/блок.
- **Копировать** — `Ctrl+C`.
- **`Пробел`** — развернуть/свернуть текущую строку (как клик по плашке справа).
- **Правый клик → контекстное меню:**
  - **Copy** — копировать выделение.
  - **Word Wrap (this line)** — для разворачиваемой строки.
  - **Open Link** — если выделен URL.
  - **Open File / Open Containing Folder** — если выделен путь к файлу.
  - **Use as Time Filter Start / End** — если выделен таймстамп.

## Фильтрация

- **Уровень лога** — кнопки на панели (Trace…Fatal) или меню `Filters → Log Level`.
- **Диапазон времени** — панель *Time Range Filter*: задайте From/To и Apply. Выделенный таймстамп можно подставить сюда из контекстного меню.
- Панель **Text Filters** — правила Include/Exclude с AND/OR, регистр и regex для каждого правила, опционально привязка к полю. Подробно — в разделе **[Text Filters и поиск](#text-filters-и-поиск)** ниже. Фильтры применяются к **активной вкладке** кнопками **Apply** / **Reset**.

## Text Filters и поиск

Панель **Text Filters** — это набор правил; каждое правило это одна карточка:

- **Contains / Not contains** — строка должна (Include) или не должна (Exclude) содержать текст.
- **AND / OR** — как правило связано с предыдущим (у первого правила связи нет).
- **Поле** — по умолчанию `(entire row)` — искать по всей строке; можно ограничить конкретной колонкой схемы Log Fields (работает, когда включён **Filter blocks**).
- **⚙ (шестерёнка)** — доп-опции правила: **Case sensitive** (учёт регистра) и **Regular expression**.
- **Образец цвета** — левый клик выбирает цвет подсветки совпадений правила, правый клик включает/выключает подсветку этого правила. Цвет автоматически подбирается контрастным к теме, но его можно переопределить.

### Как комбинируются правила (AND / OR)

Правила соединяются булевой логикой, **одинаково в обоих режимах** (см. ниже) — различается только куда попадает результат:

- **AND** («и») — строка должна удовлетворять **обоим** соседним правилам сразу. Пример: `Contains "Timeout"` **AND** `Contains "Disk"` — только строки, где в одной строке есть *и* «Timeout», *и* «Disk».
- **OR** («или») — строке достаточно удовлетворять **хотя бы одному** правилу. Пример: `Contains "Timeout"` **OR** `Contains "Disk"` — все строки с «Timeout» **плюс** все строки с «Disk».
- Приоритет: **AND связывает сильнее OR**, то есть `A AND B OR C` читается как `(A AND B) OR C`.
- **Not contains** удобно комбинировать через AND: `Contains "error"` **AND** `Not contains "timeout"` — ошибки, кроме таймаутов.

> Если несколько правил дают пустой результат — почти всегда причина в том, что они соединены **AND**, а искомые тексты находятся в **разных** строках (пересечения нет). Переключите связь на **OR**, чтобы увидеть объединение.

### Два режима: Filter и Search

Переключатель **Non-destructive search** в шапке панели:

- **Off — Filter (по умолчанию):** кнопка **Apply** *скрывает* в основном view строки, не прошедшие правила. Разрушающий для отображения (но не для файла) — быстрый способ «оставить только нужное».
- **On — Search:** кнопка **Search** оставляет основной view **полным**, а совпадения выводит списком в отдельной панели **Search Results** (снизу; открывается из `View` или сама при нажатии Search). Клик по строке результата прыгает к ней в основном view. Ничего не скрывается.

Правила AND/OR работают в обоих режимах **одинаково**. Единственное удобство: правило, добавленное *в режиме Search*, по умолчанию получает связь **OR** (чтобы «показать всё, что нашло любое правило»), а в режиме Filter — **AND** (сузить). Связь всегда можно поменять вручную.

- **Highlight in main view** — подсвечивать найденный текст и в основном view (работает в обоих режимах; строки при этом не скрываются).

### Регулярные выражения

Галочка **Regular expression** трактует текст правила как **регекс в стиле Perl/PCRE**, а не как шаблон с подстановочными знаками командной строки:

- `*` — повтор предыдущего символа, а **не** «любые символы». Чтобы задать «любые символы», пишите `.*`.
- `.` — любой один символ; `\d` — цифра; `\d+` — одна или несколько цифр.
- Правило срабатывает, если выражение найдено **где угодно** в строке (якорь не нужен).
- Примеры: `entry number \d+` найдёт «entry number 42»; `Log entry number .*` найдёт любую такую запись; `(WARN|ERROR)` — строки с любым из двух уровней.
- **Неверное выражение** подсвечивается: поле обводится красным, под ним появляется текст ошибки с позицией. Пока выражение неверно, правило не участвует в поиске.

### Профили

Комбобокс **Profile** + кнопка **⋯** сохраняют наборы правил под именами (**Save**, **Save as new…**, **Rename…**, **Delete**). При переключении профиля с несохранёнными правками программа спросит, сохранить ли их.

## Подсветка строк (Row Highlighters)

- Панель *Row Highlighters* — недеструктивно красит строки целиком по совпадению с паттерном (строки **не** скрываются). Apply/Reset действуют на активную вкладку.

## Схемы полей (Log Fields)

- Панель *Log Fields* — задайте **схему** (упорядоченные блоки: таймстамп, уровень, число, текст, regex, остаток строки…) через **Manage…** (авто-определение по строке-образцу или импорт Grok-выражения).
- Галочка **Filter blocks** показывает только выбранные блоки; выбор учитывается в **Save View As** и в текстовых фильтрах, привязанных к полям.

## Быстрый поиск по строке (панель инструментов)

- Поле поиска на панели инструментов — это разовый поиск по тексту (не путать с панелью **Text Filters** выше). `Ctrl+F` ставит фокус в него; `Enter` / `F3` — следующее совпадение, `Shift+F3` — предыдущее. Найденная строка разворачивается, термин подсвечивается.
- Для многокритериального недеструктивного поиска со списком результатов используйте режим **Search** панели **Text Filters** (см. раздел выше).

## Дочитывание файлов

- **Reload** — `F5` или кнопка на панели: дочитывает добавленное в конец содержимое.
- **Авто-дочитывание** — правый клик по кнопке Reload включает/выключает его для вкладки; интервал — в **Settings → General**.

## Панели и компоновка

- Панели включаются из меню **View** или клавишами `Ctrl+F1…F5` (Text Filters, Directory Scanner, Time Filter, Log Fields, Row Highlighters). Положение панелей запоминается между сессиями.

## Настройки и тема

- **Tools → Settings** (`Ctrl+,`):
  - **General** — расширения для сканера, интервал авто-дочитывания.
  - **Font** — моноширинный шрифт и размер (живой предпросмотр).
  - **Colors** — цвета уровней, синтаксиса, выделения и интерфейса.
  - **View** — перенос по словам по умолчанию.
  - **Shortcuts** — переназначение любой команды (есть «Restore Defaults»).
- Окно автоматически следует светлой/тёмной теме Windows.
